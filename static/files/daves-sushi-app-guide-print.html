Operations Guide

Dave's Sushi App Map

This guide is a technical walk-through of the live app: what each area is for, what staff should do in it, and how the main operational pieces connect.

Core Purpose

The app is the restaurant's central operating surface for daily prep, checklists, ordering references, menus, documents, temperature monitoring, recipes, and team notes.

How It Is Meant To Be Used

Staff move through a few focused pages during the day instead of passing paper around. Admin tools exist to maintain the system behind the scenes.

High-Level Layout

Home

The landing page. It acts as a quick-launch board into lists, docs, menu, temperatures, recipes, and admin tools.

Lists

The operational core. This area contains checklist, prep, order, and inventory flows broken into kitchen-facing sections.

Documents

Reference material such as the handbook, SOPs, and app information. This is the long-form read area.

Menu

The in-app menu viewer for both the main menu and the secret menu, including mobile-friendly zoom behavior.

Temper

The live cold-holding and sensor page. It shows the latest node readings and surfaces stale or warning states.

Whiteboard / Notes

Shared communication spaces for ideas, updates, and meeting-style notes that do not belong inside a list.

Operational Pages

Check Lists

Shift-based completion lists. Staff open the correct area, complete items, and submit the finished state for that shift.

Prep Lists

Prep-focused entries. These now support flexible text input, so staff can enter exact working notes such as cut 1, 1/3 pan, or ask kitchen.

Orders

Reference and ordering lists grouped by supplier or ordering context. These support predictable quantity-style entry.

Inventory

Area-based inventory sheets used to track counts and stock conditions with a more structured numeric workflow.

Recipes

Recipe reference area for repeatable kitchen standards and quick lookup during service or prep.

Specials

A place for special-item or service-facing content when needed without changing the core menu structure.

Documents And Visual References

Handbook / SOP

These are long-form operational references. They are rendered directly inside the app with zoom behavior tuned for phone and tablet use.

Menus

Main and secret menu pages are exposed as an in-app viewer so staff can zoom, inspect, and reference them without leaving the app.

Temperature Monitoring

1

Sensor Nodes

ESP8266-based nodes collect DHT22 readings and forward them to the gateway over ESP-NOW.

2

Gateway

The gateway receives node packets, batches recent readings, and posts them into the app's temperature ingest route.

3

Temper Page

The app turns those readings into live tiles with warning states, stale handling, and quick visual status checks.

Admin Controls

Admin / Users

Invites, account access, and role maintenance. This is where the team is admitted to or restricted from the system.

Admin / Lists

List structure maintenance. Titles, par values, references, and list items are maintained here.

Admin / Documents

Document metadata and file references for About, SOP, Handbook, and other controlled reading material.

Admin / Recipes

Recipe maintenance for any structured recipe content that should be changed without touching code.

Admin / Camera

Camera-source and capture administration for the surveillance or media side of the system.

Admin Dashboard

Moderation and oversight area for system-level activity, including whiteboard and operational review actions.

How Staff Should Work Through The App

1

Start On Home

Use Home as the jump point. The page is designed to reduce hunting and send staff to the right operational area quickly.

2

Complete The Correct List

Choose the correct checklist, prep list, order sheet, or inventory sheet for the job currently being done.

3

Use Documents As Reference, Not As Storage

Menus, SOPs, handbook pages, and this guide are meant to be referenced quickly without changing active operational data.

4

Use Notes For Communication

When something is important but does not belong in a list item, put it in the communication surfaces instead of hiding it inside a list entry.

This guide intentionally explains function and workflow without exposing internal security, credential, or infrastructure details.
